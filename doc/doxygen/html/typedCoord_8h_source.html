1 // TypedCoord.h: Definition of the CTypedCoord abstract base class.
2 //
3 // The CTypedCoord class is an abstract base class whose derived classes
4 // represents a WGS84 coordinate with a specific type. For example, a UTM
5 // coordinate class can be derived from the CTypedCoord class. These classes
6 // store a coordinate in the class' coordinate type.
7 //
8 // Since a GPS uses lat/lon format for its coordinates, and distance/bearing
9 // formulas require lat/lon coordinates, CTypedCoord contains a latitude and
10 // a longitude member variable; member functions exist for converting between
11 // the lat/lon coordinate and the class' coordinate type.
12 //
13 // Conversion of coordinates from lat/lon to another format can be very
14 // computationally intensive, especially on machines without FPUs. Because
15 // of this, it is encouraged that derived classes use lazy evaluation when
16 // returning coordinates, so the conversions take place only when necessary.
17 // Because of the lazy evaluation, derived classes should not access the
18 // coordinate member variables directly; use the GetLatLonCoord() and the
19 // SetLatLonCoord() member functions to access and/or modify these variables.
20 //
21 // Since this is an abstract base class, you *must* override the following
22 // member functions in your derived classes:
23 // - GetCoordType(): Override this member function to return the type of this
24 // object. Each derived class from CTypedCoord has a unique type ID that
25 // specified the type of this object. Add a value to the enumerated type
26 // COORD_TYPE when deriving a new class, and return this value from this
27 // function. The COORD_TYPE enumerated type is found elsewhere in this
28 // project.
29 // - CreateCoordString(): Override this member function to create a formatted
30 // coordinate std::string using the coordinate within the class.
31 // - CreateXYCoordStrings(): Override this member function to create two
32 // formatted std::strings containing the x and y coordinate values;
33 // - GetXYCoord(): Override this member function to return the (x, y)
34 // coordinate values. This function is useful when the other data (e.g.,
35 // zones, etc.) are not needed.
36 // - CreateDisplayStrings: Override this function to create four std::strings that
37 // can be used for display purposes. The four std::strings contain coordinate
38 // data; these std::strings are displayed at the top left, top right, bottom
39 // left, and bottom right of a display. For example:
40 // A lat/lon coordinate class may create std::strings as follows:
41 // - top left: North/south hemisphere character
42 // - top right: Latitude
43 // - bottom left: East/west hemisphere character
44 // - bottom right: Longitude
45 // This would create a display as follows:
46 // N 48*25'33.3"
47 // W 123*20'09.8"
48 // A UTM coordinate class may create std::strings as follows:
49 // - top left: UTM zone (e.g., "10U")
50 // - top right: UTM easting
51 // - bottom left A std::string describing the coordinate type ("UTM")
52 // - bottom right: UTM northing
53 // This would create a display as follows:
54 // 10U 475139
55 // UTM 5363695
56 // It is up to the calling function to write the std::strings to a display.
57 //
58 // Member functions that can (and should) be overridden include:
59 // - SetLatLonCoord(): Override this member function to set the current lat/
60 // lon coordinate within this class and convert the lat/lon coordinate to
61 // the appropriate coordinate type so it can be stored within this class.
62 // - GetLatLonCoord(): Override this member function so that the lat/lon
63 // coordinate can be returned by converting the class' coordinate from the
64 // class' coordinate type.
65 // - operator=(): Override this operator to implement assignment operations.
66 // Two operator=() functions should be created as follows:
67 // - An operator=() function that takes a parameter of type CTypedCoord&.
68 // Use the GetLatLonCoord() function to convert the right hand side
69 // coordinate from its current type to a pair of lat/lon values, then
70 // call this class' SetLatLonCoord() function to convert the lat/lon pair
71 // to a coordinate of this class' type.
72 // - An operator=() function that takes a parameter of a reference to an
73 // object as the same type as the object. Simply copy all member
74 // variables from the right hand side object to this object.
75 //
76 // Using coordinate objects derived from this base class, it is simple to
77 // perform such operations as:
78 // - Finding the distance between two coordinate objects with different
79 // coordinate types.
80 // - Assigning the coordinates of an object of one coordinate type to another
81 // object with a different coordinate type.
82 //
83 // [TO DO: Allow for other map datums. For now, these coordinates are using
84 // the WGS84 datum.]
85 //
86 // Written by Jason Bevins in 1998. File is in the public domain.
87 //
